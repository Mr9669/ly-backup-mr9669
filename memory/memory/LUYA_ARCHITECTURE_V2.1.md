### 1.2 九天功能定义（参考道教九天体系）

| 天名 | 方位/属性 | 功能定义 | 数据类型 | 优先级 |
|------|----------|----------|----------|--------|
| **钧天** | 中央 | 核心数据库，总控中枢 | 全维度聚合数据 | P0 |
| **苍天** | 东方/生发 | 产业生长数据，新兴行业 | 创业公司、独角兽、技术趋势 | P1 |
| **变天** | 变化/动态 | 实时变化数据，舆情监控 | 新闻、舆情、市场动态 | P0 |
| **玄天** | 北方/深邃 | 深度数据，非公开信息 | 财务、投资、内部情报 | P1 |
| **幽天** | 西南/隐秘 | 隐秘数据，情报网络 | 竞争情报、风险预警 | P2 |
| **颢天** | 西方/广阔 | 全球宏观数据，国际视野 | 地缘政治、全球经济、跨国企业 | P1 |
| **朱天** | 南方/光明 | 公开数据，知识普及 | 公开资料、科普、教育资源 | P1 |
| **炎天** | 东南/炽热 | 热点数据，趋势追踪 | 热点事件、流行趋势、风口 | P1 |
| **阳天** | 东北/初升 | 早期数据，萌芽信号 | 早期项目、实验室成果、论文 | P2 |

### 1.3 数据分级（延续三层体系）

每层数据在九天中均有分布，按敏感度分级：

```
钧天（中央调控）
    │
    ├── 第一层·开源域（免费开放）
    │   ├── 朱天（公开知识）
    │   ├── 苍天（产业概览）
    │   └── 炎天（热点追踪）
    │
    ├── 第二层·资源域（信息盈利）
    │   ├── 变天（实时舆情）
    │   ├── 颢天（全球宏观）
    │   └── 阳天（早期信号）
    │
    └── 第三层·核心域（内部使用）
        ├── 玄天（深度财务）
        ├── 幽天（隐秘情报）
        └── 钧天（核心决策）
```

### 1.4 技术实现

**数据流向：**
```
纵地金光（采集工具）
    ↓
九天数据库（分类存储）
    ↓
陆压（分析处理）
    ↓
输出应用（API/报告/决策）
```

**存储架构：**
- 钧天：核心节点，主数据库
- 其他八天：分布式节点，按功能分片
- 数据同步：实时同步关键数据至钧天
